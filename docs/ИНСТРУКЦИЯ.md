# Инструкция по работе с AutoTest Recorder & Player

Расширение для Chrome: запись и воспроизведение автотестов для веб-приложений (в том числе Angular, React и других SPA).

---

## 1. Введение

### Назначение

- **Запись сценариев** — вы выполняете действия на странице (клики, ввод, переходы), расширение сохраняет их как шаги теста.
- **Редактирование** — изменение шагов, добавление условий, циклов, API-запросов, переменных.
- **Воспроизведение** — автоматическое выполнение сохранённого теста в браузере.

Данные хранятся локально в браузере (Chrome storage). Экспорт и импорт тестов в JSON поддерживаются.

### Установка

- **Из Chrome Web Store:** найдите «AutoTest Recorder & Player» и нажмите «Установить».
- **Распакованное расширение:** откройте `chrome://extensions/`, включите «Режим разработчика», нажмите «Загрузить распакованное расширение» и укажите папку с плагином.

После установки иконка расширения появится на панели браузера. Клик по иконке открывает попап.

## 2. Режим записи

### Основной сценарий

1. Откройте попап (клик по иконке расширения).
2. При желании введите **название теста** в поле «Название теста…». Если оставить пустым, при сохранении будет подставлено имя по умолчанию (например, «Test» и дата/время).
3. Нажмите **«Начать запись»**.
4. Перейдите на вкладку с нужным сайтом и выполните действия: клики, ввод текста, выбор в выпадающих списках, переходы по ссылкам и т.д.
5. Вернитесь в попап и нажмите **«Остановить запись»**.

Тест сохраняется автоматически после остановки записи и появляется в списке «Сохранённые тесты».

### Индикаторы и кнопки в попапе

- **Статус** — отображает «Готов», «Идёт запись» или состояние воспроизведения.
- **Прогресс шагов** — при воспроизведении показывается «Шаг N из M» и тип текущего шага.
- **«Пауза» / «Продолжить»** — пауза и возобновление воспроизведения (видны во время прогона).
- **«Стоп»** — принудительная остановка воспроизведения или записи при необходимости.

### Расширенный поиск селекторов

Кнопка **«Расширенный поиск селекторов»** открывает страницу анализа селекторов для **текущей активной вкладки**. Там можно проверить, какие селекторы генерируются для элементов страницы, и использовать эту информацию при отладке тестов.

### Инспектор селекторов

Кнопка с иконкой лупы (🔍) в шапке попапа открывает **инспектор селекторов**. Он позволяет детально анализировать элементы страницы и получать для них селекторы (например, для ручной правки шагов в редакторе).

### Развернуть на весь экран

Кнопка с иконкой «развернуть» (⛶) открывает попап в **отдельном окне** (полноэкранный режим). Удобно, когда нужен постоянный доступ к списку тестов и кнопкам управления без переключения вкладок.

### Настройки записи

Кнопка с иконкой шестерёнки (⚙️) открывает **настройки** расширения.

- **Библиотеки селекторов:** включение/выключение Finder, Optimal Select, Unique Select и выбор версии (Lite / Full) для каждой. От этого зависит, как формируются селекторы при записи.
- **Приоритеты атрибутов:** порядок проверки атрибутов при генерации селекторов (например, `data-testid`, `data-cy`, `id`, `name` и т.д.). Можно менять порядок перетаскиванием.
- **Режим записи (выбор селекторов):**
  - **Автоматический** — расширение само выбирает селектор по приоритетам.
  - **С выбором селектора (пикер)** — при каждом клике во время записи показывается панель с вариантами селекторов; вы выбираете нужный.
  - **Через инспектор** — открывается полный инспектор селекторов для детального анализа.

В настройках пикера (при режиме «С выбором селектора») можно задать таймаут автовыбора (секунды), включить отображение оценок стабильности и выделение лучшего варианта.

При использовании функций, связанных с GigaChat (улучшение селекторов, генерация данных и т.д.), потребуется настройка учётных данных и при необходимости локального прокси. Подробнее см. [README_PROXY.md](../README_PROXY.md) в корне проекта.

#### Получение Access token (GigaChat API)

Чтобы отправлять авторизованные запросы к GigaChat API, нужен **Access token**.

- **Запрос для получения токена:** `POST /api/v2/oauth` (полный URL при прямом подключении: `https://ngw.devices.sberbank.ru:9443/api/v2/oauth`). В теле передаётся параметр `scope=GIGACHAT_API_PERS`, в заголовке `Authorization: Basic <base64(client_id:client_secret)>`.
- **Срок действия:** токен действует **30 минут**. После истечения его нужно запрашивать снова.
- **Поведение в расширении:** токен запрашивается **только при необходимости** — перед первым обращением к API (анализ селектора, проверка подключения, генерация данных и т.д.) и при истечении кэшированного токена. Если в данный момент запросов к GigaChat нет, запрос за токеном не выполняется. Кэш токена хранится в памяти; при перезагрузке расширения токен запрашивается заново при первом обращении к API.

### Запись видео при воспроизведении

В настройках можно включить **запись видео** во время прогона теста. Файлы сохраняются в папку «Загрузки» → `AutoTestRecorder/videos/` в формате WebM. По умолчанию файл меньше по размеру, но без перемотки в стандартном плеере. Включите опцию **«Делать видео перематываемым (постобработка)»**, чтобы после записи автоматически добавлять индекс перемотки — тогда файл станет чуть больше, зато перемотка будет работать в любом плеере. Без этой опции для перемотки можно открыть файл в **VLC** или конвертировать в MP4: `ffmpeg -i файл.webm -c copy файл.mp4`

## 3. Режим редактирования

### Открытие редактора

В попапе в блоке **«Сохранённые тесты»** у каждой карточки теста есть кнопка **«Редактировать»** (✏️). По нажатию открывается **редактор теста** в новой вкладке (`editor/editor.html?testId=…`).

### Шапка редактора

- **Название теста** — поле для изменения имени теста.
- **«Сохранить»** — сохранение изменений в хранилище расширения.
- **«Экспорт JSON»** — выгрузка теста в JSON-файл.
- **«Импорт JSON»** — загрузка теста из JSON-файла (замена или добавление).
- **«Импорт API»** — импорт сценария из спецификации OpenAPI/Swagger (YAML/JSON): выбор эндпоинтов и опции (тестовые данные, валидация ответов, сохранение в переменные).

### Режимы запуска из редактора

- **«Полный прогон»** — выполнение всех шагов теста подряд.
- **«Режим отладки»** — пошаговое выполнение (удобно для отладки).
- **«Параллельный прогон»** — выполнение в нескольких вкладках (если поддерживается сценарием).
- **«Оптимизированный прогон»** — появляется после успешной оптимизации теста (если функция доступна).

### Список действий (шаги теста)

- **Добавление:** кнопка **«Добавить»** с выпадающим меню: добавить действие, условие, цикл.
- **Быстрые шаги:** кнопка **«Быстрые шаги»** — шаблоны: Telegram-сообщение, Webhook, REST GET, REST POST.
- **Редактирование шага:** кнопка «Редактировать» (✏️) у шага — открывается модальное окно с полями действия (тип, селектор, значение и т.д.).
- **Удаление:** кнопка удаления у шага.
- **Перетаскивание** — шаги можно менять местами перетаскиванием (если интерфейс поддерживает drag-and-drop).
- **Группировка** — кнопка «Группировать» объединяет шаги по страницам (URL).
- **«Свернуть все»** — сворачивание/разворачивание списка шагов.
- **«Показать URL»** / **«Показать наименование поля»** — отображение или скрытие URL шагов и подписей полей.

### Переменные

Панель **«Переменные»** (кнопка «📊 Переменные» в шапке) управляет переменными сценария.

- **Добавить вручную** — кнопка «➕ Добавить»: имя переменной, значение, флаги «глобальная», «чувствительные данные».
- **Извлечение из URL** — кнопка «🔗 Из URL»: выбор URL из шагов теста или текущей страницы, тип извлечения (query-параметры, сегменты пути, регулярное выражение), имя переменной.
- **«Со страницы»** — выбор элемента на открытой вкладке и сохранение его текста (или части) в переменную; при необходимости — извлечение по регулярному выражению.
- **«Из localStorage страницы»** — выбор вкладки, загрузка списка ключей localStorage и сохранение выбранного значения в переменную теста.
- **Фильтры переменных:** «Все», «Пустые», «Для данного сценария», «Глобальные», «Входные», «Выходные», «В теле запроса» — для удобного просмотра списка.

Переменные используются в шагах через подстановки (например, `{var:имя}`). В редакторе можно задать слоты переменных для шагов и редактировать значения переменных.

### Условия и циклы

- **Условие** — добавляется через «Добавить» → «Условие». Редактируются ветки «then» и «else» и выражения условий.
- **Цикл** — добавляется через «Добавить» → «Цикл». Редактируется тело цикла и параметры (количество итераций или условие).

Внутри условий и циклов можно добавлять обычные шаги (действия).

### Импорт API (OpenAPI/Swagger)

Кнопка **«Импорт API»** в шапке открывает модальное окно:

1. Выбор файла спецификации (YAML или JSON).
2. После загрузки отображаются эндпоинты; можно выбрать нужные (галочки).
3. Опции: генерация тестовых данных, валидация ответов, сохранение ответов в переменные, базовый URL (если не указан в спецификации).
4. Кнопка **«Импортировать выбранные»** создаёт шаги теста по выбранным эндпоинтам.

### Незаданные переменные

При запуске теста (полный прогон, отладка и т.д.) если в шагах используются переменные, для которых не задано значение, открывается диалог **«Незаданные переменные»**: список переменных и кнопки «Задать переменные», «Продолжить» или «Отмена». Задав значения, можно сохранить их и снова запустить тест.

## 4. Воспроизведение

### Из попапа

1. В списке «Сохранённые тесты» найдите нужный тест.
2. Нажмите кнопку **«Запустить»** (▶️) на карточке теста.
3. Откройте или обновите вкладку с целевым сайтом (рекомендуется обновить страницу перед запуском).
4. Во время прогона в попапе отображаются «Пауза», «Продолжить» и «Стоп». Используйте их при необходимости.

Воспроизведение выполняется на активной вкладке (или в той, которую укажет расширение). Прогресс шагов отображается в попапе.

### Из редактора

В шапке редактора доступны кнопки:

- **«Полный прогон»** — запуск всего теста.
- **«Режим отладки»** — пошаговое выполнение.
- **«Параллельный прогон»** — запуск в нескольких вкладках (если применимо).
- **«Оптимизированный прогон»** — при наличии после оптимизации.

Перед запуском рекомендуется **обновить страницу** на целевом сайте, чтобы тест выполнялся в предсказуемом состоянии.

## 5. Дополнительно

### История прогонов

На карточке теста в попапе кнопка **«История»** открывает модальное окно с историей прогонов: дата/время, успех/ошибка, при необходимости детали по шагам. Удобно для анализа стабильности теста и поиска ошибок.

### Скриншоты

Кнопка **«Скриншоты»** на карточке теста открывает отдельную страницу просмотра скриншотов, сделанных во время прогонов (по шагам и прогонам). Доступна, если при воспроизведении было включено сохранение скриншотов.

### Экспорт в Excel

В попапе при наличии кнопки или пункта меню **экспорта в Excel** можно выгрузить отчёт по тестам и прогонам (результаты, шаги, ошибки) в файл Excel. Файл сохраняется через стандартный диалог сохранения браузера (или в папку «Загрузки»).

### Импорт теста

Кнопка **«Импорт»** (📥) в блоке «Сохранённые тесты» открывает выбор JSON-файла. После выбора файла тест импортируется и появляется в списке. При большом количестве тестов учтите ограничения хранилища (см. блок «Память»).

### Память (хранилище)

Внизу попапа отображается блок **«Память»**: использование хранилища расширения (общее и по категориям: тесты, история, переменные, настройки, скриншоты, файлы). Кнопка «▼» разворачивает детали и кнопки очистки:

- **Очистить историю** — удаление истории прогонов.
- **Очистить скриншоты** — удаление сохранённых скриншотов.
- **Очистить файлы** — удаление загруженных файлов (если используются).
- **Очистить всё** — полная очистка данных расширения (тесты, история, переменные, настройки, скриншоты, файлы). Используйте осторожно.

### Настройки

Через кнопку настроек (⚙️) в попапе доступны:

- Настройки генерации селекторов (библиотеки, приоритеты атрибутов).
- Режим записи (автоматический, с выбором селектора, через инспектор) и параметры пикера.
- Настройки GigaChat (учётные данные, проверка подключения).
- При необходимости — настройки прокси для GigaChat (подробнее в [README_PROXY.md](../README_PROXY.md)).

## Краткая справка

| Действие | Где |
|----------|-----|
| Начать запись | Попап → «Начать запись» |
| Остановить запись | Попап → «Остановить запись» |
| Редактировать тест | Попап → карточка теста → «Редактировать» |
| Запустить тест | Попап → карточка теста → «Запустить» или Редактор → «Полный прогон» |
| Импорт теста | Попап → «Импорт» (📥) |
| Экспорт теста | Редактор → «Экспорт JSON» |
| История прогонов | Попап → карточка теста → «История» |
| Скриншоты | Попап → карточка теста → «Скриншоты» |
| Настройки | Попап → иконка шестерёнки (⚙️) |

При возникновении ограничений по количеству тестов или объёму данных рекомендуется удалить неиспользуемые тесты или очистить историю/скриншоты через блок «Память» в попапе.
