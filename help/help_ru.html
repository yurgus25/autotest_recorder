Запись тестов

Расширение фиксирует: клики, ввод текста, выбор из <select> и кастомных dropdown, переходы, навигацию.

Режимы выбора селекторов

Настраиваются в ⚙️ → Режим записи:

Автоматический

Расширение само выбирает лучший селектор по приоритетам атрибутов. Подходит для большинства случаев.

Пикер

При каждом клике показывается панель с вариантами селекторов — вы выбираете нужный.

Инспектор

Полный инспектор элемента с детальным анализом всех доступных селекторов.

Для генерации используются библиотеки Finder, Optimal Select и Unique Select (включаются в настройках, для каждой есть версия Lite/Full).

💡 Совет: Перед записью обновите страницу. Действуйте без спешки — расширение фиксирует каждый клик. Имя теста можно задать в popup до остановки записи.

Редактор тестов

Открывается кнопкой ✏️ Редактировать в карточке теста.

Условия и циклы

Условие — содержит ветки «then» и «else» с произвольным выражением.
Цикл — задаётся количеством итераций или условием выхода.
Внутри условий и циклов размещаются обычные шаги.

Переменные

Кнопка 📦 Переменные в редакторе. Переменные позволяют переиспользовать данные между шагами.

Создание переменных

➕ ВручнуюИмя, значение, флаги «глобальная», «чувствительные данные».
🔗 Из URLИзвлечение из URL шага: query-параметр, путь, регулярное выражение.
📄 Со страницыВыбор элемента на вкладке и сохранение его текста.
🗄️ Из localStorageПолучение значения из localStorage страницы.

Области видимости

В модальном окне переменных есть вкладки:

📄 СценарийПеременные текущего теста.
🔁 ЦиклыПеременные внутри циклов.
🔀 УсловияПеременные внутри условий.
🌐 ГлобальныеДоступны во всех тестах.

Подстановка в шагах

Используйте {var:имя_переменной} в полях значения, URL, заголовках и теле API-запросов.

Фильтры (в панели переменных)

Все · Пустые · Для данного сценария · Глобальные · Входные · Выходные · В теле запроса.

💡 Если при запуске теста есть переменные без значения — откроется диалог для их заполнения.

Воспроизведение

Запуск: ▶️ в popup или режим запуска в редакторе.

  • Прогресс — в popup: «Шаг N из M» и тип текущего действия.
  • ⏸️ Пауза и ⏹️ Стоп доступны в любой момент.
  • Результат сохраняется в 📊 Историю теста.

Скриншоты

При включённой опции расширение снимает экран при воспроизведении. Сохранение на диск (папка «Загрузки») и/или в памяти расширения для просмотра через 📸. Можно ограничить «только при ошибках».

Видео

При включённой опции записывается видео вкладки в формате WebM. Опция «Постобработка» добавляет индекс перемотки. Файлы сохраняются в подпапку videos внутри папки сохранения.

Импорт из OpenAPI / Swagger

Кнопка 🌐 Импорт API в редакторе.

1

2
Выберите нужные эндпоинты галочками.
3

«Импортировать выбранные» — шаги добавятся в тест.

Настройки

Открываются через ⚙️ в popup. Кнопки внизу страницы: Сбросить, Экспорт, Импорт, Сохранить.

🎯 Генерация селекторов

Библиотеки — Finder, Optimal Select, Unique Select (вкл/выкл, версия Lite/Full).

Приоритеты атрибутов — порядок проверки: data-testid, data-cy, id, name и др. Меняется перетаскиванием.

🎬 Режим записи

Выбор селекторов — Автоматический / Пикер / Инспектор.

Стратегия — Стабильность / Читаемость / Кратчайший.

Пикер — таймаут автовыбора, показ оценок стабильности, выделение лучшего варианта.

⚡ Производительность

Кэширование селекторов, максимальная длина селектора, таймаут генерации.

📊 Аналитика

Включает дашборд: графики успешности, проблемные шаги, heatmap ошибок. В popup появляется кнопка 📊.

🎬 Видео и 📸 Скриншоты

Видео — запись вкладки в WebM, постобработка для перемотки.

Скриншоты — на диск и/или в память расширения. Опция «только при ошибках».

Папка — задаётся относительно «Загрузок» (по умолчанию AutoTestRecorder). Внутри создаются подпапки videos и screenshots.

🚀 Оптимизация селекторов

Группы настроек по фазам:

  • 🔴 Фаза 1 — дедупликация, кэширование, умная оценка стабильности, штраф за динамические части.
  • 🟠 Фаза 2 — улучшенное обнаружение dropdown, экспоненциальный backoff при ретраях, MutationObserver.
  • 🟡 Фаза 3 — генерация XPath, поиск оверлеев, валидация перед сохранением, debounce/throttle событий.
  • 🟢 Метрики — расчёт метрик стабильности, автоисправление нестабильных селекторов, порог стабильности.

📁 Файлы для тестов

Загрузка файлов для использования в шагах (например, при загрузке документов через формы).

📊 Экспорт в Excel

Выгрузка отчётов в Excel/CSV/XLS. Опции: автоэкспорт, триггеры (запись/прогон/оптимизация), логирование ошибок консоли, формат файла, разделитель, путь сохранения.

🔧 Расширенные

Включение/отключение автотестов, подробное логирование, исключение скрытых элементов, умные ожидания.
